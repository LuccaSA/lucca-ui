luifNumber

A filter useful when displaying numbers aligned to the right in a table and some of them have non integer part but some don't and nothing is aligned and it's horrible

It is a slight modification of the filter number provided by angular but it puts the non integral part in an invisible span if this part is zero, allowing it to be correctly aligned in a table of values with text-align: right

Also, this page uses the locale FR-fr for no other reason than to show that the luifNumber supports the same locale as angular i18n

luifStartFrom

A filter for ng-repeat if you want to start from a certain index and don't want to use ng-if="$index >= myIndex"

Angular provides a version of this filter but for the end, see https://docs.angularjs.org/api/ng/filter/limitTo

A typical usage is in the case of paginating a list ng-repeat="item in list | luifStartFrom : myIndex | limitTo : 20"

Usage:

<ANY ng-repeat="val in values | luifStartFrom : 2">...</ANY>
<ANY ng-repeat="val in values | luifStartFrom : myIndex">...</ANY>
<ANY ng-repeat="val in values | luifStartFrom : myIndex | limitTo : cnt">...</ANY>

Demo:

myIndex:

all the values ng-repeat="val in values"

  • $index: {{$index}} - value: {{val}}

all the values starting from myIndex ng-repeat="val in values | luifStartFrom : myIndex"

  • $index: {{$index}} - value: {{val}}

all the values starting from myIndex paginated to display only five elements ng-repeat="val in values | luifStartFrom : myIndex | limitTo : 5"

  • $index: {{$index}} - value: {{val}}

luifMoment

A filter to display a date, a moment or a string using the formats provided by momentjs

Will try to transform the input to a moment, if the resulting moment is not valid it will just return the input

if none is provided, will use the format LLL

Usage:

{{ myValue | luifMoment : "LL" }}
{{ myValue | luifMoment : myFormat }}

Demo:

myValue: (enter '2015-01-01' or such like, or try something that is not parsable by moment)

set myValue to

new Date()
moment().startOf('week')

myFormat:

myValue formatted: {{ myValue | luifMoment : myFormat }}

luifCalendar

A filter to display a date, a moment or a string using the function moment().calendar()

Will try to transform the input to a moment, if the resulting moment is not valid it will just return the input

You can pass a reference date as an argument to the filter. If none is provided or moment cannot parse it, it will use today

Does not support the formats arguments available from momentjs version 2.10.5 onwards

Usage:

{{ myValue | luifCalendar }}
{{ myValue | luifCalendar : myRefDate }}

Demo:

myValue:

set myValue to

start of week
last week

myRefDate:

myValue relative to today: {{ myValue | luifCalendar }}

myValue relative to myRefDate: {{ myValue | luifCalendar : myRefDate }}

luifFriendlyRange

A filter to display a block having a start and an end, the start and end may be the same day/year/month and you dont want to repeat the same information so you can't just use start.format('LL') - end.format('LL')

Will try to find property start, startsOn, startsAt, end, endsOn, endsAt

If both start's and end's year is the current year, it will not be displayed because saying "September 25th - October 2nd" implies "September 25th - October 2nd, this year"

It has the parameter excludeEnd for when you are dealing with block going from start included to end excluded. Example : september is from September 1st included to October 1st excluded or from Septembre 1st included to September 30th included

Usage:

{{ myBlock | luifFriendlyRange }}
{{ myBlock | luifFriendlyRange : true }}

Demo:

myBlock.start:

myBlock.end:

myBlock: {{ myBlock | luifFriendlyRange }}

myBlock with end excluded: {{ myBlock | luifFriendlyRange : true }}

luifDuration

A filter to display a timespan or a moment.duration

Will try to transform the input to a moment.duration, if the resulting duration is not valid, it will return nothing

You can provide a format (see below) but if none is provided it will try to display

Important: the argument format does not work for durations over 24 hours because moment.duration does not support .format(). It will likely disappear in an incoming version in favor of an argument "mainUnit" with the logic being 35h is

  • 1d 11h for 'day' as main unit
  • 35h for 'hour' as main unit
  • 2100m for 'minute' as main unit
Associated issue

Usage:

{{ myValue | luifDuration }}
{{ myValue | luifDuration : myFormat : showSign }}

Demo:

myValue: (enter '1.11:30:00' or 'PT35H30m' or such like)

set myValue to

35h
-30m

myValue: {{ myValue | luifDuration }}

myValue with format = HH: {{ myValue | luifDuration : 'HH' }} - try entering a duration over 24h to see it doesn't work

myValue with a sign: {{ myValue | luifDuration : '' : true }}

luifHumanize

A filter to cast the input to a moment.duration and call the method humanize on it

the first argument is to use the suffix in the humanize function

  • moment.duration(3,'m').humanize(): 3 minutes
  • moment.duration(3,'m').humanize(true): in 3 minutes

Usage:

{{ myValue | luifHumanize }}
{{ myValue | luifHumanize : usePrefix }}

Demo:

myValue: (enter '1.11:30:00' or 'PT35H30M' or such like)

set myValue to

1h
-30m

myValue humanized: {{ myValue | luifHumanize }}

myValue humanized with a suffix: {{ myValue | luifHumanize : true }}
